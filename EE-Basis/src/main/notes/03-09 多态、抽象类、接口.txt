020_多态
	什么是多态？
		同一个对象在不同时刻展示出来的多种形态

	多态的前提？
		1.有继承/实现关系			inherit or implements 关系
		2.有方法重写  			Override
		3.父类引用指向子类对象		接口或者父类 标识符 = new 实现类名

	使用多态时，访问成员变量的特点？
		编译看左边，运行看左边
	使用多态时，访问成员方法的特点？
		编译看左边，运行看右边

022_多态的好处和弊端
	多态的好处？	提高程序的扩展性
					定义方法时，使用父类类型作为参数，在使用的时候，使用具体的子类类型参与操作
	多态的弊端？	不能使用子类的特有成员（包括成员变量和特有方法）

026_抽象类
	被abstract修饰的类叫抽象类

	1.有抽象方法的类必须是抽象类，这句话是否正确？        对
	2.抽象类必须有抽象方法，这句话是否正确？             错
	3.子类继承一个抽象父类，是否需要重写父类的全部抽象方法？
	答：不需要，但只有重写了父类的全部抽象方法，才可以不被定义为抽象子类

030_接口
	生活中的接口是什么意思?
		就是一种规范

	java中的接口是什么意思?
		更多的是对行为的抽象
		也是一种规范--------规范的是它的实现类应该有什么！！！

031_接口特点
	接口的定义格式？
		public interface 接口名{
			//抽象方法;
		}
	实现类实现接口的格式？
		public class 实现类名 implements 接口名{
			//重写/实现抽象方法
		}
	接口能不能实例化？
		不能！！

032_接口的成员特点
	接口的成员变量有什么特点？------实质是常量
		默认修饰符： public static final
	接口中能不能有构造方法？------不能
	        抽象类中可以后构造方法，只用于实现类的对象生成
	接口中能不能有非抽象的成员方法？------不能
	思考：
		接口中可以有什么？
			常量；
			抽象方法；
